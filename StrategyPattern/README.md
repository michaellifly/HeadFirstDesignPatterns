# 策略模式
### 定义
策略模式定义了算法族，分别封装起来，让它们之间可以互相替换，此模式让算法独立于使用算法的客户。
### 设计原则
* 找出应用中可能需要变化之处，把它们独立出来，不要和那些不需要变化的代码混在一起。
* 针对接口编程，而不是针对实现编程
* 多用组合，少用继承
### 使用场景
* 实现某一个功能有多种方案可以选择
* 需要在不同时间应用不同的业务规则
